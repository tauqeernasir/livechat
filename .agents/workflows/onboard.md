Perform a quick onboarding pass for this repository.

Required workflow:
1. Read and use these project docs first:
   - [Product requirements](../../PRD.md)
   - [Engineering guideline](../../GUIDELINE.md)
2. Scan the current codebase structure and key implementation files.
3. Use Git to summarize recent work:
   - Review the latest commits (for example, recent 5-10 commits).
   - Highlight newly added/modified files and what changed at a high level.
4. Output a short onboarding summary only:
   - Project purpose (2-4 bullets)
   - Current architecture/status (2-4 bullets)
   - Recent changes from Git (2-5 bullets)
   - Immediate risks or unknowns (optional, max 3 bullets)

Constraints:
- Keep the summary concise and practical.
- If no code exists yet, explicitly say so and base the summary on docs + git history.
